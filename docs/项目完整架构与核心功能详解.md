# 1. 项目概览

## 1.1 这是什么项目？

一个 **AI 驱动的广告推荐 Android 应用**，核心亮点：

| 功能 | 说明 |
|------|------|
| 🏠 **信息流浏览** | 全屏滑动广告列表，支持推荐/关注/热门三个 Tab 切换 |
| 🔍 **对话式 AI 搜索** | 微信聊天风格的自然语言搜索，AI 理解意图后推荐广告 |
| 📄 **广告详情页** | 大图/视频全屏预览 + AI 深度分析 + 互动操作 |
| ❤️ **互动系统** | 点赞、收藏、评论，数据持久化到 Room 数据库 |
| 📊 **埋点统计** | 曝光/点击数据记录 + 实时点击率计算 |
| 🎬 **视频播放** | 基于 ExoPlayer，支持静音切换和详情页全屏播放 |
| 🎨 **启动动画** | "让广告更懂你"六个字逐字弹跳入场 |

## 3.1 入口：AndroidManifest.xml

```xml

<activity android:name=".ui.screen.SplashActivity"
          android:exported="true">
    <intent-filter>
        <action android:name="android.intent.action.MAIN" />
        <category android:name="android.intent.category.LAUNCHER" />

</activity>

<!-- 设置了自定义 Application 类 -->
<application android:name=".AiAdApplication" ...>
```

用户点击桌面图标 → 系统启动 `SplashActivity`。

但是在这之前，**Application 先初始化**：

## 3.2 第一步：`AiAdApplication.onCreate()`

```java
// AiAdApplication.java
public class AiAdApplication extends Application {
    private AppDatabase database;

    @Override
    public void onCreate() {
        super.onCreate();
        // 创建 Room 数据库实例（全局单例，整个App生命周期只有一个）
        database = Room.databaseBuilder(
            getApplicationContext(),
            AppDatabase.class,
            AppDatabase.DATABASE_NAME  // "ai_ad_recommendation.db"
        ).build();
    }

    // 供其他类获取数据库实例
    public AppDatabase getDatabase() { return database; }
}
```

**作用**：在 App 最早期创建 Room 数据库的全局单例。之后任何 Activity/Fragment 通过 `((AiAdApplication) getApplication()).getDatabase()` 获取数据库实例，避免重复创建。

**调用链**：系统创建 Application → `onCreate()` → `Room.databaseBuilder(...).build()` → 数据库就绪。

## 3.3 第二步：`SplashActivity.onCreate()`

```
SplashActivity → 播放启动动画 → 自动跳转到 MainActivity → finish()
```

### 核心方法：`bounceLetter(View view)`

```java
private void bounceLetter(View view) {
    // 1. 上下弹跳：从 0 向上偏移 -45px，再弹回 0
    ObjectAnimator upDown = ObjectAnimator.ofFloat(view, "translationY", 0f, -45f, 0f);
    upDown.setDuration(600);  // 600ms
    upDown.setInterpolator(new OvershootInterpolator(1.5f));  // 超过目标再弹回

    // 2. 水平缩放：1.0 → 1.3 → 1.0（先放大再恢复）
    ObjectAnimator scaleX = ObjectAnimator.ofFloat(view, "scaleX", 1f, 1.3f, 1f);
    scaleX.setDuration(600);

    // 3. 垂直缩放
    ObjectAnimator scaleY = ObjectAnimator.ofFloat(view, "scaleY", 1f, 1.3f, 1f);
    scaleY.setDuration(600);

    // 4. 三个动画同时播放
    AnimatorSet set = new AnimatorSet();
    set.playTogether(upDown, scaleX, scaleY);
    set.start();
}
```

**执行时序**：
```
t=0ms      → 等待 400ms
t=400ms    → '让' 字弹跳 (600ms)
t=560ms    → '广' 字弹跳 (160ms间隔)
t=720ms    → '告' 字弹跳
t=880ms    → '更' 字弹跳
t=1040ms   → '懂' 字弹跳
t=1200ms   → '你' 字弹跳
t=1800ms   → 动画结束 + 500ms停顿
t=2300ms   → 跳转到 MainActivity，finish()自身
```

**用到的 Android 知识点**：
- `ObjectAnimator.ofFloat(view, propertyName, from, to, ...)` — 属性动画
- `OvershootInterpolator(1.5f)` — 弹簧插值器，值1.5表示超过目标50%再弹回
- `AnimatorSet.playTogether(...)` — 同时播放多个动画
- `Handler.postDelayed(runnable, delay)` — 延迟执行

## 4.2 广告实体：`AdItem`

```java
public class AdItem implements Serializable {
    private String id;              // 广告ID，如 "ad_001"
    private String title;           // 标题
    private String description;     // 描述
    private String imageUrl;        // 图片地址，指向 assets/media/images/
    private String videoUrl;        // 视频URL，指向 res/raw/
    private String type;            // 卡片类型：large_image / small_image / video
    private String advertiser;      // 广告主
    private String category;        // 分类：推荐 / 关注 / 热门

    private String tags;            // 标签，逗号分隔，如 "科技,新品,热卖"
    private int impressions;        // 曝光次数
    private int clicks;             // 点击次数
```

| 字段 | 类型 | 用途 |

| `description` | String | 详情页完整描述文本 |
| `imageUrl` | String | 图片地址，`file:///android_asset/...` 格式加载本地资源 |
| `videoUrl` | String | 视频地址，`android.resource://...` 格式加载 raw 资源 |

| `category` | String | Tab 分类：推荐 / 关注 / 热门 |
| `summary` | String | AI 摘要，显示在卡片描述位置 |
| `tags` | String | 逗号分隔的标签，用于AI匹配和标签展示，如 "科技,新品" |

## 4.3 用户行为实体：`UserBehavior`

```java
@Entity(tableName = "user_behaviors")
public class UserBehavior {
    @PrimaryKey(autoGenerate = true) private long id;  // 自增主键
    private String adId;           // 关联的广告ID
    private String behaviorType;   // 行为类型：like / favorite / comment / impression / click
    private String commentText;    // 评论内容（仅 behaviorType=comment 时有值）
    private long timestamp;        // 时间戳
}
```

**5种行为类型**（定义在 `Constants.java`）：

| 常量 | 值 | 含义 | 触发时机 |
|------|-----|------|----------|
| `BEHAVIOR_LIKE` | `"like"` | 点赞 | 用户点击❤按钮 |
| `BEHAVIOR_FAVORITE` | `"favorite"` | 收藏 | 用户点击⭐按钮 |
| `BEHAVIOR_COMMENT` | `"comment"` | 评论 | 用户发表评论 |
| `BEHAVIOR_IMPRESSION` | `"impression"` | 曝光 | 广告卡片进入可见区域 |
| `BEHAVIOR_CLICK` | `"click"` | 点击 | 用户点击广告卡片进入详情页 |

> **设计要点**：5 种行为共用一个表，通过 `behaviorType` 字段区分，而不是建 5 张表。
> - 点赞和收藏通过"存在即激活"判定：有一条记录 = 已激活，删除记录 = 取消
> - 曝光和点击只增不删，每次触发插入新记录
> - 评论存储 `commentText` 字段

## 4.4 行为 DAO：`BehaviorDao`

```java
@Dao
public interface BehaviorDao {
    // 查某广告某类型行为的数量，如"这个广告有多少人点赞"
    @Query("SELECT COUNT(*) FROM user_behaviors WHERE ad_id = :adId AND behavior_type = :type")
    LiveData<Integer> getBehaviorCount(String adId, String type);

    // 查某广告的所有评论
    @Query("SELECT * FROM user_behaviors WHERE ad_id = :adId AND behavior_type = 'comment' ORDER BY timestamp DESC")
    LiveData<List<UserBehavior>> getComments(String adId);

    // 同步查询版本（仅在后台线程使用！toggle 逻辑用）
    @Query("SELECT * FROM user_behaviors WHERE ad_id = :adId AND behavior_type = :type LIMIT 1")
    UserBehavior getBehaviorSync(String adId, String type);

    // 插入
    @Insert(onConflict = OnConflictStrategy.REPLACE)
    void insertBehavior(UserBehavior behavior);

    // 删除（取消点赞/收藏时）
    @Delete
    void deleteBehavior(UserBehavior behavior);
}
```

**两种查询方法的区别**：
- `getBehavior(...)` → 返回 `LiveData<UserBehavior>`，在主线程观察，数据变更自动通知 UI
- `getBehaviorSync(...)` → 返回 `UserBehavior`（同步），**只能在后台线程调用**，用于 toggle 逻辑中判断当前状态

## 4.7 Mock 数据源：`MockDataSource`

这是本项目的核心数据源，从 `assets/mock/ad_feed.json` 读取 30 条模拟广告数据。

### `loadMockData()` — 读取 JSON 文件

```java
public void loadMockData() {
    // 1. 打开 assets/mock/ad_feed.json
    InputStream is = context.getAssets().open(Constants.MOCK_DATA_FILE);

    // 2. 逐行读取整个文件 → StringBuilder
    BufferedReader reader = new BufferedReader(new InputStreamReader(is, StandardCharsets.UTF_8));
    StringBuilder sb = new StringBuilder();
    String line;
    while ((line = reader.readLine()) != null) {
        sb.append(line);
    }
    reader.close();

    // 3. Gson 把 JSON 字符串 → List<AdItem>
    Type listType = new TypeToken<List<AdItem>>() {}.getType();
    allAds = gson.fromJson(sb.toString(), listType);
}
```

> 注：`TypeToken<List<AdItem>>(){}.getType()` 是 Gson 解析泛型的标准写法（Java 泛型擦除导致无法直接 `List<AdItem>.class`）。

### `getPage(int page)` — 分页获取

```java
public List<AdItem> getPage(int page) {
    if (allAds == null) loadMockData();  // 懒加载

    int start = page * Constants.PAGE_SIZE;  // PAGE_SIZE = 10
    if (start >= allAds.size()) return new ArrayList<>();  // 空列表 = 没有更多了

    int end = Math.min(start + Constants.PAGE_SIZE, allAds.size());
    return new ArrayList<>(allAds.subList(start, end));
    //              ↑ 包装一层避免 ConcurrentModificationException
}
```

**分页逻辑**：
- page=0 → start=0, end=10 → 返回第1~10条
- page=1 → start=10, end=20 → 返回第11~20条
- page=3 → start=30 ≥ 30 → 返回空列表 → ViewModel 设 hasMore=false

### `getPageByCategory(String category, int page)` — 按分类+分页

```java
public List<AdItem> getPageByCategory(String category, int page) {
    if (allAds == null) loadMockData();

    // 1. Stream 过滤：只保留 category 匹配的广告
    List<AdItem> filtered = allAds.stream()
        .filter(ad -> ad.getCategory().equals(category))
        .collect(Collectors.toList());

    // 2. 在过滤后的列表中分页
    int start = page * Constants.PAGE_SIZE;
    if (start >= filtered.size()) return new ArrayList<>();
    int end = Math.min(start + Constants.PAGE_SIZE, filtered.size());
    return new ArrayList<>(filtered.subList(start, end));
}
```

### `getRandomAds(int count)` — 随机获取（下拉刷新用）

## 4.8 行为数据仓库：`BehaviorRepository`

```java
public class BehaviorRepository {
    private final BehaviorDao behaviorDao;
    // 单线程线程池 — 保证所有写操作串行执行（避免数据库并发问题）
    private final ExecutorService executor = Executors.newSingleThreadExecutor();
}
```

### `toggleLike(String adId, OnToggleCallback callback)` — 切换点赞

```java
public void toggleLike(String adId, OnToggleCallback callback) {
    executor.execute(() -> {  // 在后台线程执行
        // 1. 查数据库：当前有没有点赞记录？
        UserBehavior existing = behaviorDao.getBehaviorSync(adId, BEHAVIOR_LIKE);

        if (existing != null) {
            // 2a. 有 → 删除（取消点赞）
            behaviorDao.deleteBehavior(existing);
            callback.onResult(false);   // false = 现在是"未点赞"
        } else {
            // 2b. 没有 → 插入新记录（点赞）
            UserBehavior like = new UserBehavior(adId, BEHAVIOR_LIKE, null, System.currentTimeMillis());
            behaviorDao.insertBehavior(like);
            callback.onResult(true);    // true = 现在是"已点赞"
        }
    });
}
```

**Toggle 模式**：用一条方法处理"开"和"关"，内部通过查数据库判断当前状态：
- 有记录 → 删除 → 回调 `false`（已取消）
- 无记录 → 插入 → 回调 `true`（已激活）

### 为什么用 `ExecutorService`？

Room 不允许在主线程执行数据库写入操作，会抛异常。所以所有 insert/delete 操作都通过 `executor.execute(...)` 放到后台线程。注意：
- 用 `newSingleThreadExecutor()` 保证串行 → 避免"一个线程删、一个线程插"的并发冲突
- LiveData 的查询自动在后台线程执行，不用手动切换

### 状态变量（UI 观察的 LiveData）

### `loadFirstPage()` — 首次加载

```java
public void loadFirstPage() {
    isLoading.setValue(true);                // 1. 开始 loading
    List<AdItem> data = repository.loadInitialAds();  // 2. 从 Repository 拿数据
    adList.setValue(data);                   // 3. 更新列表 → UI 自动刷新
    currentPage.setValue(0);                 // 4. 重置页码
    hasMore.setValue(data.size() >= PAGE_SIZE); // 5. 判断是否有下一页
    isLoading.postValue(false);              // 6. 结束 loading（postValue 线程安全）
}
```

> `setValue` vs `postValue`：`setValue` 必须在主线程调用，`postValue` 可以在任何线程调用（内部切换到主线程）。

### `loadNextPage()` — 上滑加载更多

```java
public void loadNextPage() {
    // 1. 防重入检查
    if (isLoading.getValue() || !hasMore.getValue()) return;

    isLoading.setValue(true);

    int nextPage = currentPage.getValue() + 1;
    List<AdItem> newData;

    // 2. 根据当前Tab选择数据源
    if ("推荐".equals(currentTab.getValue())) {
        newData = repository.loadNextPage(nextPage);      // 推荐：分页加载
    } else {
        newData = repository.loadAdsByCategory(tab, nextPage); // 分类：按分类+分页
    }

    // 3. 合并新旧数据
    if (newData.isEmpty()) {
        hasMore.setValue(false);  // 没有更多了
    } else {
        List<AdItem> merged = new ArrayList<>(adList.getValue());
        merged.addAll(newData);
        adList.setValue(merged);
        currentPage.setValue(nextPage);
    }

    isLoading.postValue(false);
}
```

**上滑加载更多触发时机**：`RecyclerView` 滚动监听中，`lastVisible >= totalItems - 2` 时（距离底部还有2条时预加载）。

### `switchTab(String tab)` — Tab 切换

```java
public void switchTab(String tab) {
    currentTab.setValue(tab);        // 更新当前Tab
    isLoading.setValue(true);

    List<AdItem> data;
    if ("推荐".equals(tab)) {
        data = repository.loadInitialAds();    // 推荐：全量加载首页
    } else {
        data = repository.loadAdsByCategory(tab, 0); // 关注/热门：按分类加载
    }

    adList.setValue(data);           // 替换列表（不是追加）
    currentPage.setValue(0);         // 页码重置
    hasMore.setValue(data.size() >= PAGE_SIZE);
    isLoading.postValue(false);
}
```

### `refresh()` — 下拉刷新

```java
public void refresh() {
    isRefreshing.setValue(true);

    List<AdItem> newData;
    if ("推荐".equals(currentTab.getValue())) {
        newData = repository.refreshAds();  // 推荐：随机10条
    } else {
        newData = repository.loadAdsByCategory(tab, 0);  // 分类：重新加载
        Collections.shuffle(newData);                     // 也打乱顺序
    }

    if (newData != null && !newData.isEmpty()) {
        adList.setValue(newData);
        currentPage.setValue(0);
    }
    isRefreshing.setValue(false);
}
```

**刷新逻辑**：不是清空重载，而是用 `getRandomAds()` 换一批随机的新数据。

```java
        if (matched.isEmpty()) {
```

```java
private List<AdItem> matchByIds(String adIds) {
```

这是最复杂的方法，按以下步骤将广告数据绑定到 UI：

```
1. 设置标题、描述(=AI摘要)、广告主文本
2. Glide 加载图片（item.getImageUrl() → ImageView，centerCrop）
3. 色块作为加载过程中的占位背景（按category显示不同颜色）
4. 判断广告类型：
   ├─ video → 显示封面图+▶播放按钮 → 点击跳详情页（auto_play=true）
   └─ 非video → 隐藏播放按钮
5. 动态生成标签 chip（遍历 tags 逗号分隔，每个tag是一个 TextView + 圆角背景）
6. 观察点赞状态（LiveData → InteractionBar.setLiked）
7. 观察收藏状态（LiveData → InteractionBar.setFavorited）
8. 观察评论数（LiveData → InteractionBar.setCommentCount）
9. 点赞点击 → behaviorRepo.toggleLike(adId)
10. 收藏点击 → behaviorRepo.toggleFavorite(adId)
11. 评论点击 → 跳转详情页
12. 卡片整体点击 → 跳转详情页（Intent传AdItem）
```

### `formatNumber(int num)` — 数字格式化

```java
private String formatNumber(int num) {
    if (num >= 10000) return String.format("%.1fw", num / 10000f);  // 12580 → "1.3w"
    if (num >= 1000) return String.format("%.1fk", num / 1000f);    // 3420 → "3.4k"
    return String.valueOf(num);                                       // 520 → "520"
}
```

# 9. UX 细节：动画与交互

## 9.1 启动页动画

| 参数 | 值 | 说明 |
|------|-----|------|
| 初始停顿 | 400ms | 给用户看清画面的时间 |
| 字间间隔 | 160ms | 每个字依次触发 |
| 单字时长 | 600ms | 每个字的弹跳+缩放 |
| 弹跳距离 | -45px (上) | `translationY: 0 → -45 → 0` |
| 缩放幅度 | 1.0 → 1.3 → 1.0 | `scaleX` + `scaleY` |
| 插值器 | `OvershootInterpolator(1.5f)` | 弹簧效果，超过目标50%弹回 |
| 总时长 | ~2300ms | 6个字都跳完 + 500ms 停顿 → 跳转 |

## 9.2 点赞动画

```java
// 心形缩放动画，300ms，OvershootInterpolator
ValueAnimator scaleX = ValueAnimator.ofFloat(1.0f, 1.3f, 1.0f);
ValueAnimator scaleY = ValueAnimator.ofFloat(1.0f, 1.3f, 1.0f);
// 两个动画同时播放 → 心跳般的感觉
```

## 9.3 视频播放器行为

| 场景 | 行为 |
|------|------|
| 信息流卡片 | 不创建 ExoPlayer，显示封面图+▶按钮（避免闪烁和性能问题） |

# 10. 如何扩展与替换

## 10.1 替换 Mock 数据为真实 API

1. 修改 `AdRepository`，将 `MockDataSource` 替换为 Retrofit API 调用
2. `loadInitialAds()` → API 调用首页接口
3. `loadNextPage(page)` → API 调用下一页接口
4. `refreshAds()` → API 调用刷新接口
5. ViewModel 层 **完全不需要修改**（Repository 隔离的好处）

## 10.2 接入真实大模型

当前已集成 DeepSeek API，要换其他模型：
- 修改 `AiApiService.API_KEY` / `API_URL` / `MODEL`
- 或者改为通义千问、ChatGPT 等（接口兼容 OpenAI 格式）

## 10.3 添加新的 Tab 分类

1. 在 Mock 数据的 `category` 字段添加新分类
2. 在 `MainFeedFragment.onViewCreated()` 添加 `tabLayout.addTab(tabLayout.newTab().setText("新分类"))`
3. `MainFeedViewModel.switchTab()` 中不需改（走 `loadAdsByCategory` 通用路径）

## 10.4 添加新的行为类型

1. 在 `Constants` 添加新常量（如 `BEHAVIOR_SHARE = "share"`）
2. 在 `BehaviorDao` 添加查询方法
3. 在 `BehaviorRepository` 添加操作方法
4. UI 层调用新方法即可
