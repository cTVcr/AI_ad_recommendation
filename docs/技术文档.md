# AI 广告推荐信息流 — 技术文档

> 个人参赛 | Android Native Java | MVVM 架构 | 训练营课题

---

## 一、项目概述

实现一个单列 AI 广告推荐信息流 App。用户上下滑动浏览广告卡片，点赞/收藏/评论，AI 对话搜索推荐广告。

### 技术栈

| 层级 | 技术 |
|------|------|
| 语言 | Java 11 |
| UI | XML布局 + RecyclerView/ViewPager2 |
| 架构 | MVVM (ViewModel + LiveData) |
| 数据库 | Room (SQLite 封装) |
| 网络 | Retrofit2 + OkHttp |
| 视频 | ExoPlayer (media3) |
| AI | DeepSeek API (OpenAI 兼容格式) |

### 设计原则

- **单一数据源**：Room 是所有数据的唯一真相源，两个页面读同一张表
- **可复用组件**：`InteractionBar` 在主界面卡片和详情页复用，`VideoPlayerView` 独立封装
- **面向接口**：`AiService` 接口定义了抽象契约，Mock 和 API 两种实现可互换

## 三、核心功能实现

### 3.1 数据流：MVVM 架构

```
用户操作 → Fragment/Activity → ViewModel → Repository → 数据源
                                   ↑                       │
                                   └─── LiveData(观察) ────┘
```

**我学到的东西**：
- Activity/Fragment 只负责 UI 展示和事件转发，不存业务数据
- ViewModel 通过 LiveData 把数据"推"给 UI，不需要手动刷新
- Repository 是门面模式，把数据来源(Mock/API/数据库)的差异藏起来
- 屏幕旋转时 ViewModel 不销毁，数据不丢失 —— 这就是 MVVM 相比传统 MVC 的核心优势

### 3.2 无限信息流 + 下拉刷新

**原理**：RecyclerView 滑动监听 + 分页截取

```
PAGE_SIZE = 10，总共12条
首次: getPage(0) → [0-9]      ← 10条
滑到底: getPage(1) → [10-11]   ← 2条
滑到底: getPage(2) → []        ← 到底了
```

**下拉刷新**：调用 `getRandomAds()` → 打乱顺序 → 重置页码 → UI 刷新

**当前方案**：SwipeRefreshLayout + RecyclerView.OnScrollListener
**未来升级**：Paging3 库(自动处理加载状态、重试、缓存)

### 3.3 三种卡片样式（多 ViewType）

**原理**：RecyclerView 通过 `getItemViewType()` 返回不同值，根据值加载不同布局

| viewType | 样式 | 布局文件 |
|----------|------|----------|
| 0 | 大图/视频模式 | `view_ad_card.xml` |
| 1 | 小图模式 | `item_ad_small_image.xml` |

**当前方案**：色块(不同分类不同颜色) + 叠加标题，不依赖网络图片
**未来升级**：Glide 加载真实图片/CDN 图片

### 3.4 视频播放器

**设计决策**：主页不创建播放器(防闪烁)，详情页全屏有声播放

```
主页视频卡片:  色块 + ▶按钮
               ↓ 点 ▶
详情页:       setupVideo() → ExoPlayer → 有声全屏
               ← 返回 → onDetachedFromWindow → releasePlayer()
```

**当前方案**：ExoPlayer，从 `res/raw` 加载本地视频
**未来升级**：CacheDataSource 边播边缓存，第二个播放秒开
**为什么主页不播放**：RecyclerView 回收 item 时 SurfaceView 重建会闪，类似抖音只在详情页播放

### 3.5 点赞/收藏/评论 + 内外同步

**核心机制**：Room (SQLite) + LiveData 自动同步

```
                  Room 数据库 (user_behaviors 表)
                       ↙           ↘
         主界面 Adapter             详情页
    getLikeCount().observe()   likeCount.observe()
                       ↘           ↙
                    数据变了 → 自动刷新UI
```

**流程**：用户点赞 → Room 写入 → 数据库发通知 → 所有观察者收到 → UI 更新

**当前方案**：Room + LiveData，同一张表，两边自动同步
**未来升级**：线上用户系统 + Firestore/后端同步

### 3.6 埋点统计（曝光 + 点击率）

**曝光记录**：RecyclerView 滑动时，当前可见的卡片 ID 加入 Set，调用 `recordImpression()`
**点击记录**：进入详情页 → `recordClick()`
**去重**：`java.util.HashSet<String>` 保存已曝光 ID，避免重复计数
**点击率**：`点击数 / 曝光数 × 100%`，两个 `LiveData.observe()` 都到齐后计算

### 3.7 AI 对话搜索

**两个模式**：

| 模式 | 条件 | 实现 |
|------|------|------|
| 大模型模式 | 填了 API Key | Retrofit 调 DeepSeek API，返回 JSON `{"reply":"...", "ids":["ad_001","ad_006"]}` |
| Mock 模式 | 没填 API Key | `MockAiService` 关键词 → 标签映射表 → 标签匹配广告 |

**大模型模式流程**：
```
用户输入"手机"
  → 构造12条广告列表文本发给 AI
  → System prompt: "你是广告推荐助手。广告库: [ad_001]华为... [ad_006]小米..."
  → AI 分析后返回 JSON: {"reply":"为你找到2款手机","ids":["ad_001","ad_006"]}
  → 按 ID 取广告 → 聊天气泡展示
```

**当前方案**：DeepSeek API (OpenAI 兼容格式) + 本地 Mock 兜底
**未来升级**：任何兼容 OpenAI 格式的模型都可替换(Qwen/GPT/Claude)

### 3.8 详情页 AI 分析 + 换一个

```
详情页打开 → loadAiAnalysis()
  → API: "你是广告分析专家。标题:华为Mate 60 Pro。描述:搭载麒麟芯片..."
  → 返回格式: 👥 目标人群 / 💡 卖点亮点 / 🎯 购买建议
  → 用户点 [⟳ 换一个] → 重新调 API(temperature=0.7 每次不同)
```

**当前方案**：大模型生成 → 分析卡片展示 → 换一个按钮
**未来升级**：带历史记录的分析，避免每次相同

## 四、各功能方案对比

### 4.1 信息流列表

| | 当前方案 | 备选方案 | 对比 |
|---|---------|---------|------|
| 列表控件 | RecyclerView | ViewPager2 | RecyclerView 更灵活，支持多 ViewType |
| 图片加载 | 无(色块) | Glide | 色块免网络，真项目应用 Glide+CDN |
| 下拉刷新 | SwipeRefreshLayout | 自定义 | 原生组件，一行代码 |
| 分页 | 手动计算 subList | Paging3 | Paging3 更健壮但学习成本高 |

### 4.2 数据持久化

| | 当前方案 | 备选方案 | 对比 |
|---|---------|---------|------|
| 数据库 | Room | 手写 SQLiteOpenHelper | Room 编译时检查 SQL，自动生成代码 |
| 状态同步 | LiveData | EventBus/回调 | LiveData 生命周期安全，自动取消订阅 |
| 数据量 | 本地 12 条 Mock | 后端 API 分页 | 架构已预留，Repository 换数据源即可 |

### 4.3 视频播放

| | 当前方案 | 备选方案 | 对比 |
|---|---------|---------|------|
| 播放器 | ExoPlayer | MediaPlayer | ExoPlayer 更灵活，支持 DASH/HLS/缓存 |
| 主页列表 | 色块+按钮(不放) | 放播放器 | 不放 = 不闪, 抖音同款方案 |
| 缓存 | 无(预留扩展点) | CacheDataSource | 架构支持，上线时加一行配置 |
| 播放源 | res/raw 本地 | 网络 URL | 本地秒开，演示效果好 |

### 4.4 AI 搜索

| | 当前方案 | 备选方案 | 对比 |
|---|---------|---------|------|
| AI 分析方式 | AI直接分析12条广告选ID | AI返回标签→本地匹配 | 直接选更准确("手机"必出华为+小米) |
| API | DeepSeek(Retrofit) | 任意兼容 OpenAI 的模型 | 架构解耦，改 URL+Key 即可切换 |
| 兜底 | Mock 关键词匹配 | 无兜底 | 没有 API 也能跑通完整流程 |
| 对话 | 单轮上下文+历史 | 长对话记忆 | 每轮带最近3轮历史，保持上下文 |

### 4.5 UI 风格

| | 当前方案 | 备选方案 | 对比 |
|---|---------|---------|------|
| 背景 | 暖米色 #F9F6F0 | 深色/纯白 | 暖色柔和 |
| 卡片 | 白色 16dp 圆角+阴影 | 直角/Material3 | 精致不僵硬 |
| 色块 | 马卡龙色系(蓝/粉/橙/绿) | 灰色/单色 | 按分类区分，一眼识别 |
| 图标 | VectorDrawable | PNG | 矢量无限缩放，包体积小 |

## 五、项目架构流程图

### 整体启动流程

```
App 启动 → SplashActivity (1.5秒弹跳动画)
         → MainActivity → MainFeedFragment
                            ├─ TabLayout (推荐/关注/热门)
                            ├─ RecyclerView (无限信息流)
                            │   ├─ 大图卡片
                            │   ├─ 小图卡片
                            │   └─ 视频卡片(▶按钮)
                            │       → DetailActivity (全屏播放)
                            └─ 搜索框
                                → SearchDialogFragment (聊天弹窗)
```

### 数据流

```
MockDataSource (assets/ad_feed.json)
    → AdRepository
        → MainFeedViewModel (LiveData<List<AdItem>>)
            → MainFeedFragment → Adapter → 12张卡片

用户操作(点赞/评论/点击):
    → BehaviorRepository → Room 写入
        → LiveData 通知 → 主界面+详情页双向更新
```

### AI 搜索流

```
用户输入 "推荐手机"
    → SearchViewModel.sendMessage()
        → AiChatHelper.analyzeAndMatch()
            ├─ 有API Key → DeepSeek API (返回广告ID)
            └─ 无API Key → MockAiService (标签匹配)
        → matchByIds() → 聊天气泡展示
```

## 八、答辩要点

### 架构设计
> "采用 MVVM + Repository 模式，View 层只做展示，ViewModel 管理状态并通过 LiveData 驱动UI更新，Repository 作为数据门面隐藏了数据来源细节。Room 作为唯一数据源实现主界面和详情页的内外数据同步。"

### 数据同步
> "点赞/收藏状态的同步用了 Room 的 LiveData 机制。两边都通过 `getLikeCount(adId).observe()` 观察同一条数据库查询，任意一方写入 Room 自动推送给所有观察者，无需手动回调或 EventBus。"

### 视频播放
> "ExoPlayer 封装为 VideoPlayerView 组件。主页显示静态封面+播放按钮避免 RecyclerView 中 SurfaceView 闪烁问题，点按跳详情页初始化播放器。`onDetachedFromWindow` 自动释放资源防止内存泄漏。"

### AI 搜索
> "搜索采用大模型直接分析广告库的方案：将 12 条广告的标题/标签/描述发给 DeepSeek，AI 直接返回匹配的广告 ID 列表和自然语言推荐理由，比标签匹配更精准。同时有本地 Mock 兜底，未配置 API Key 时自动降级为关键词匹配。"

### 零基础学习
> "从 Android 零基础到完成这个项目，我掌握了 MVVM 架构、Room 数据库、ExoPlayer 视频播放、Retrofit 网络请求、RecyclerView 多 ViewType、自定义 View 组件化、LiveData 数据同步、大模型 API 集成等十多项核心技术。项目从架构设计到组件封装到功能实现均为个人独立完成。"
